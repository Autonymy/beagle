# Plan: documentation consolidation

## Problem

~7,300 lines across 55+ doc files. The form catalog is duplicated 5
times. The type system is documented 4 times with inconsistent names.
The tools list is copy-pasted between CLAUDE.md and Scribble. Adding a
form or changing a type requires touching 5+ files manually. The
"canonical source" (Scribble) isn't what actually gets edited.

Current `beagle-docs-sync` handles 3 mechanical facts (test count,
stdlib count, devlog count). Everything else is manual.

## Diagnosis

The root cause: **no clear ownership boundary between code-derived facts
and prose.** Forms, types, stdlib entries, and tools are defined in
Racket source, but docs treat them as hand-authored prose that must be
manually kept in sync across files.

## Architecture: code is the source of truth

```
Racket source (types.rkt, parse.rkt, stdlib-*.rkt, bin/*)
        │
        ▼
  beagle-docs-sync (extract + template)
        │
        ├──► docs/cheatsheet.md       (developer LLM reference)
        ├──► docs/cheatsheet-consumer.md  (consumer agent reference)
        ├──► CLAUDE.md                 (contributor instructions — facts injected)
        └──► scribblings/*.scrbl       (human HTML docs — facts injected)
```

**Principle:** If a fact is defined in code, no doc file should
hand-maintain it. docs-sync extracts and injects. Prose around the
facts is hand-authored but lives in exactly one place per audience.

## The three audiences

| Audience | File | Purpose | Size target |
|----------|------|---------|-------------|
| LLM developer agent | `docs/cheatsheet.md` | System context for writing beagle | ~400 lines |
| LLM consumer agent | `docs/cheatsheet-consumer.md` | Minimal "how to read/write beagle" | ~150 lines |
| Human contributor | `CLAUDE.md` + Scribble | Architecture, design decisions, workflow | CLAUDE.md ~200 lines |

Everything else is either generated, deleted, or archived.

## What to consolidate

### `docs/prompts/contributors/src.md` → merge into CLAUDE.md

Currently 137 lines that duplicate CLAUDE.md's architecture and

reference. One file, one place.

### `docs/prompts/` directory → simplify

After deletions:
```
docs/prompts/
└── README.md   (3 lines: "cheatsheet.md is the developer prompt,
                  cheatsheet-consumer.md is the consumer prompt")
```

Or delete `docs/prompts/` entirely and note in CLAUDE.md that the
cheatsheets double as agent system prompts.

### CLAUDE.md → strip language facts, keep contributor instructions

CLAUDE.md currently has ~120 lines of language reference (form lists,
stdlib counts, type catalogs) that duplicate cheatsheet.md. Strip these
to a summary line + pointer:

```markdown
## Language reference

See `docs/cheatsheet.md` for the full language reference.
Current: X forms, Y stdlib entries, targets: clj/cljs/js/nix/py.
```

The forms list, stdlib breakdown, and target feature descriptions
belong in the cheatsheet, not in CLAUDE.md. CLAUDE.md keeps:
architecture, design decisions, setup, tools, devlog discipline,
portability rule — things a contributor needs that a consumer doesn't.

## What to enhance in `beagle-docs-sync`

### Phase 1: template markers

Add `{{marker}}` support to docs-sync. When docs-sync runs, it replaces
markers with extracted content. Markers are fenced so hand-authored
prose around them is preserved:

```markdown
## Types

<!-- BEGIN:types -->
Primitives: `Int`, `Float`, `Bool`, `String`, `Nil`, `Any`, `Keyword`, `Symbol`
<!-- END:types -->
```

docs-sync extracts the primitive list from `private/types.rkt` and
injects between the markers. Hand-authored prose before/after the
markers is untouched.

### Phase 2: form catalog extraction

Extract the form list from `parse.rkt` struct definitions:

```bash
grep 'struct.*#:transparent' private/parse.rkt
```

Generate a markdown table of all forms with their AST node names.
Inject into cheatsheet.md between markers.

### Phase 3: stdlib catalog extraction

Already partially done (counts). Extend to generate per-target function
lists. This replaces `docs/stdlib-target-report.md` with a generated
artifact.

### Phase 4: tools list extraction

Extract from `bin/` directory + the `--help` output of each tool.
Inject into CLAUDE.md and Scribble between markers. Eliminates the
copy-paste problem.

### Phase 5: Scribble type name fix

Have docs-sync rewrite deprecated type names (`Long`→`Int`,
`Double`→`Float`, `Boolean`→`Bool`) in Scribble files. This is the
one place where docs-sync writes to Scribble rather than markdown.
Or: just fix Scribble once by hand and add a CI check that deprecated
names don't reappear.

## Implementation order

### Step 1: Delete the dead weight

- Delete `docs/forms.md` (1102 lines — cheatsheet.md + Scribble cover it)
- Delete `docs/cheatsheet-distilled.md` (87 lines — consumer cheatsheet exists)
- Delete `docs/findings.md` (465 lines — superseded by experiments/report.md)
- Archive `docs/agent-corrections.md` (repair pool abandoned E15)
- Replace `docs/prompts/consumers/full.md` with 3-line pointer to cheatsheet.md
- Replace `docs/prompts/consumers/distilled.md` with 3-line pointer
- Merge `docs/prompts/contributors/src.md` into CLAUDE.md (or delete if already covered)

### Step 2: Strip CLAUDE.md language facts

Remove the ~120 lines of form/type/stdlib catalogs from CLAUDE.md.
Replace with summary + pointer to cheatsheet.md. Keep: architecture,
design decisions, tool list, setup, devlog discipline, portability rule.

### Step 3: Add template markers to cheatsheet.md

Add `<!-- BEGIN:types -->` / `<!-- END:types -->` blocks around the
type list, form summary, and stdlib counts. Hand-authored prose stays
outside the markers.

### Step 4: Enhance docs-sync with marker support

Extend `bin/beagle-docs-sync` to:
1. Parse `<!-- BEGIN:X -->` / `<!-- END:X -->` fences
2. Extract facts from source (types from types.rkt, stdlib counts, etc.)
3. Replace content between markers
4. Leave everything else untouched

### Step 5: Fix Scribble type names

One-time manual fix: `Long`→`Int`, `Double`→`Float`, `Boolean`→`Bool`
in all `.scrbl` files. Add a docs-sync check that errors (not warns)
if deprecated names reappear.

### Step 6: Generate stdlib-target-report

Make `docs/stdlib-target-report.md` a generated artifact (or delete it
and put the essential info in cheatsheet.md behind markers).

## What NOT to do

- Don't try to generate markdown FROM Scribble. Scribble's toolchain
  emits HTML, not markdown. Fighting this is wasted effort.
- Don't make Scribble the "source" for LLM context. Markdown is better
  for that use case. Scribble stays as the human HTML docs source.
- Don't build a complex templating engine. `<!-- BEGIN -->` / `<!-- END -->`
  markers with sed/awk replacement is sufficient.
- Don't consolidate devlog/ or experiments/ — those are well-organized
  and serve their purpose.
- Don't delete docs/agent-workflow.md — it's unique content (decision
  tree), not a duplicate.

## Success metric

After a form addition, the contributor runs:
1. Add form (parse/check/emit/test)
2. Add one entry to cheatsheet.md (prose, between markers if generated)
3. `bin/beagle-docs-sync` propagates mechanical facts everywhere else

Two manual edits + one command. Not five files + hoping you didn't miss one.
